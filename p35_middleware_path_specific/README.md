# About project
1. In this project we are creating middleware


# About Middleware
1. A middleware is a function that intercept every HTTP request and response in you FastAPI application
2. It allows you to run code before a request is processed by a specific endpoint and after a response is generated, enabling tasks like adding headers, logging, or measuring performance.
3. The middleware runs for every HTTP request to any endpoint(e.g. if you add more endpoints, the middleware will still execute for them)
4. Middleware applies globally to all HTTP request(not just specific endpoints) unless filtered by logic inside the middleware(e.g., checking the request URL).

# Creating Middleware
1. Middleware created using the `@app.middleware("http")` decorator on an async function
2. The Middleware function takes request: The incomming HTTP request
3. The call_next: A function that passes the request to the next layer(path operation) and returns the response

# When Middleware is executed
1. Before the request reaches the path operation(endpoint)
2. After the response is generated by the endpoint, before it's returned to the client

# Middleware order
1. middleware will execute buttom to top for request
2. middleware will execute top to buttom for response
